# 03 — Query Engine

> Raw bytes on disk are not useful until you can ask questions. This phase turns SQL strings into executed operations.

---

## 🎯 Objective

By the end of this phase you will have:

- A **Lexer** that tokenizes SQL strings into meaningful tokens
- A **Parser** that builds an Abstract Syntax Tree (AST) from tokens
- An **Executor** that walks the AST and calls the storage and index layers
- Support for: `INSERT INTO`, `SELECT *`, `SELECT ... WHERE id = N`
- A REPL (Read-Eval-Print Loop) as the user interface

---

## 🧠 Concepts

---

### Concept 1: The Query Processing Pipeline

Every database processes a query through a fixed pipeline. No shortcuts.

```
SQL String
    │
    ▼
┌─────────┐
│  Lexer  │  Tokenizes: "SELECT * FROM users" → [SELECT, *, FROM, IDENT("users")]
└────┬────┘
     │
     ▼
┌──────────┐
│  Parser  │  Builds AST: SelectStmt(table="users", columns=["*"], where=None)
└────┬─────┘
     │
     ▼
┌──────────────┐
│   Planner    │  Decides HOW to execute: use index? full scan? (simplified here)
└──────┬───────┘
       │
       ▼
┌──────────────┐
│   Executor   │  Calls storage/index, collects results, returns rows
└──────────────┘
```

In a production database (PostgreSQL), each stage is deeply sophisticated. Your planner will be simple (if WHERE clause has indexed column, use index; otherwise full scan), but the architecture is identical.

---

### Concept 2: Lexical Analysis (Tokenization)

**What**: Breaking a raw string into *tokens* — atomic units of meaning.

**Analogy**: Reading a sentence. "The quick brown fox" → ["The", "quick", "brown", "fox"]. Punctuation and whitespace are delimiters, not tokens.

**SQL tokens you need:**

| Token Type | Examples |
|------------|---------|
| `KEYWORD` | SELECT, INSERT, INTO, FROM, WHERE, VALUES, AND |
| `IDENTIFIER` | users, id, name, age |
| `INTEGER` | 42, 0, 9999 |
| `STRING` | 'alice', 'hello world' |
| `STAR` | * |
| `COMMA` | , |
| `LPAREN` | ( |
| `RPAREN` | ) |
| `EQUALS` | = |
| `SEMICOLON` | ; |
| `EOF` | (end of input) |

**The lexer's job**: consume characters left-to-right, produce a flat list of tokens. The parser figures out grammar — the lexer just classifies.

**How real databases do it**: PostgreSQL's lexer is `scan.l` (Lex/Flex grammar file, ~2000 lines). It handles Unicode, escape sequences, dollar-quoting, and dozens of edge cases. Yours will handle ASCII and the basics.

---

### Concept 3: Parsing and the AST

**What**: Taking a flat list of tokens and building a tree that represents the *structure* of the query.

**Why a tree?**: SQL is hierarchical. `SELECT a FROM t WHERE x = 1 AND y = 2` has a top-level SELECT node with children: column list, table name, and a WHERE subtree containing an AND node with two comparison children.

**Your AST nodes:**

```python
@dataclass
class SelectStmt:
    table: str
    columns: list[str]       # ["*"] or ["id", "name"]
    where: 'WhereClause | None'

@dataclass
class InsertStmt:
    table: str
    values: list[int | str]  # positional, schema-matched

@dataclass
class WhereClause:
    column: str
    op: str                  # "=" for now
    value: int | str
```

**Parsing strategy**: Recursive descent. Each grammar rule becomes a function. The parser looks at the next token (lookahead) to decide which branch to take.

```
Grammar (simplified):
  statement    → select_stmt | insert_stmt
  select_stmt  → SELECT column_list FROM IDENT where_clause?
  insert_stmt  → INSERT INTO IDENT VALUES LPAREN value_list RPAREN
  column_list  → STAR | IDENT (COMMA IDENT)*
  where_clause → WHERE IDENT EQUALS literal
  literal      → INTEGER | STRING
```

**How PostgreSQL does it**: PostgreSQL's parser is generated by `gram.y` (Yacc/Bison grammar, ~17,000 lines). It produces a generic `Node *` tree. The planner then analyzes the tree with full catalog lookups, expression simplification, and join planning. Your recursive descent parser is a stripped-down but identical in concept version.

---

### Concept 4: The Query Executor

**What**: Walks the AST and performs the actual operations.

**Two execution paths:**

**INSERT path:**
```
InsertStmt
    │
    ├─ Serialize values → bytes (RecordSerializer)
    ├─ Find the table's current data page (or allocate one)
    ├─ Call Page.add_record(bytes)
    ├─ Write page to disk
    └─ If table has an index: BTree.insert(key, RecordPointer)
```

**SELECT path (with index):**
```
SelectStmt + WhereClause(column=id, op='=', value=42)
    │
    ├─ If 'id' is indexed: BTree.search(42) → RecordPointer
    ├─ DiskManager.read_page(ptr.page_id)
    ├─ Page.get_record(ptr.slot_id)
    └─ RecordSerializer.deserialize(bytes) → row tuple
```

**SELECT path (full scan):**
```
SelectStmt + no WHERE (or WHERE on non-indexed column)
    │
    ├─ For each data page in the table:
    │     raw = DiskManager.read_page(page_id)
    │     page = Page.from_bytes(page_id, raw)
    │     For each slot in page:
    │         record = page.get_record(slot_id)
    │         if not deleted: yield deserialize(record)
    └─ Return all matching rows
```

---

### Concept 5: The Catalog

A database must know the schema of every table: what columns exist, what types they have, which column is the primary key, which page is the first data page.

This information is called the **catalog** (PostgreSQL calls it `pg_catalog`). It must itself be stored durably on disk.

**Minimal catalog for your engine:**

```python
# Stored as a JSON string (or serialized records) in a special metadata page

{
  "users": {
    "schema": [["id", "int"], ["name", "str"], ["age", "int"]],
    "primary_key": "id",
    "index_root_page": 3,
    "first_data_page": 4
  }
}
```

For simplicity, store the catalog as a JSON string in page 1 of the database file (page 0 is the file header). You will outgrow this and want a proper B-Tree-backed catalog eventually, but JSON on a page is perfectly fine for this phase.

---

### Concept 6: The REPL

Read-Eval-Print Loop. The user interface. Reads a line, evaluates it, prints the result, loops.

```python
while True:
    try:
        query = input("db> ").strip()
        if not query:
            continue
        if query.lower() in ('quit', 'exit'):
            break
        result = engine.execute(query)
        for row in result:
            print(row)
    except Exception as e:
        print(f"Error: {e}")
```

This is all `main.py` needs to be. The engine does the real work.

---

## 🧩 System Design

```
main.py (REPL)
    │  engine.execute(sql_string)
    ▼
Engine (facade class — ties everything together)
    │
    ├─ Lexer.tokenize(sql_string) → list[Token]
    ├─ Parser.parse(tokens) → AST Node
    └─ Executor.execute(ast_node) → list[tuple]
         │
         ├─ Catalog (schema + page metadata)
         ├─ DiskManager
         ├─ Page
         ├─ RecordSerializer
         └─ BTree
```

The `Engine` class is the only thing `main.py` knows about. Everything else is an implementation detail.

```python
from dataclasses import dataclass

    SELECT = auto()
    INSERT = auto()
    INTO = auto()

    VALUES = auto()

    # Symbols
    STAR = auto()
    COMMA = auto()
    LPAREN = auto()
    RPAREN = auto()
    EQUALS = auto()
    SEMICOLON = auto()
    # Literals
    INTEGER = auto()
    STRING = auto()    # 'single quoted'
    IDENT = auto()     # table/column names
    # Control
    EOF = auto()

    type: TokenType

    'values': TokenType.VALUES,
    'and': TokenType.AND,

class Lexer:
    def __init__(self, text: str):
        self.text = text
        self.pos = 0

    def tokenize(self) -> list[Token]:
        tokens = []
        while self.pos < len(self.text):
            self._skip_whitespace()
            if self.pos >= len(self.text):
                break

                tokens.append(token)
        tokens.append(Token(TokenType.EOF))
        return tokens
```

### Task 3.2 — Parser (60–90 min)

Create `src/query/parser.py`.

Implement a recursive descent parser. The key pattern:

```python
class Parser:
    def __init__(self, tokens: list[Token]):
        self.tokens = tokens
        self.pos = 0

    def _current(self) -> Token:
        return self.tokens[self.pos]

    def _consume(self, expected_type: TokenType = None) -> Token:
        tok = self._current()
        if expected_type and tok.type != expected_type:
            raise SyntaxError(f"Expected {expected_type}, got {tok.type} at position {self.pos}")
        self.pos += 1
        return tok

    def _peek(self) -> Token:
        return self.tokens[self.pos]

    def parse(self) -> SelectStmt | InsertStmt:
        tok = self._peek()
        if tok.type == TokenType.SELECT:
            return self._parse_select()
        elif tok.type == TokenType.INSERT:
            return self._parse_insert()
        else:
            raise SyntaxError(f"Unknown statement starting with {tok.type}")

    def _parse_select(self) -> SelectStmt:
        self._consume(TokenType.SELECT)
        # parse column list: * or ident, ident, ...
        # consume FROM
        # consume table name (IDENT)
        # optionally parse WHERE clause
        # return SelectStmt(...)
        ...

    def _parse_insert(self) -> InsertStmt:
        self._consume(TokenType.INSERT)
        self._consume(TokenType.INTO)
        # consume table name
        # consume VALUES
        # consume ( value, value, ... )
        # return InsertStmt(...)
        ...

    def _parse_where(self) -> WhereClause | None:
        if self._peek().type != TokenType.WHERE:
            return None
        self._consume(TokenType.WHERE)
        # consume column name
        # consume =
        # consume literal value
        # return WhereClause(...)
        ...
```

### Task 3.3 — Catalog (30–45 min)

Create `src/storage/catalog.py`.

The catalog lives in **page 1** of the database file. Stored as UTF-8 JSON, zero-padded to `PAGE_SIZE`.

```python
import json

class Catalog:
    CATALOG_PAGE_ID = 1

    def __init__(self, disk_manager):
        self.dm = disk_manager
        self.tables = {}
        self._load()

    def _load(self):
        raw = self.dm.read_page(self.CATALOG_PAGE_ID)
        # Strip trailing null bytes
        text = raw.rstrip(b'\x00').decode('utf-8')
        if text:
            self.tables = json.loads(text)

    def _save(self):
        text = json.dumps(self.tables)
        data = text.encode('utf-8')
        if len(data) > PAGE_SIZE:
            raise RuntimeError("Catalog too large for one page")
        padded = data + b'\x00' * (PAGE_SIZE - len(data))
        self.dm.write_page(self.CATALOG_PAGE_ID, padded)

    def create_table(self, name: str, schema: list, pk_column: str) -> dict:
        if name in self.tables:
            raise ValueError(f"Table {name} already exists")
        # Allocate an index root page and a first data page
        index_root = self.dm.allocate_page()
        first_data = self.dm.allocate_page()
        self.tables[name] = {
            'schema': schema,
            'primary_key': pk_column,
            'index_root_page': index_root,
            'first_data_page': first_data,
            'data_pages': [first_data],
        }
        self._save()
        return self.tables[name]

    def get_table(self, name: str) -> dict:
        if name not in self.tables:
            raise ValueError(f"Table {name} does not exist")
        return self.tables[name]
```

### Task 3.4 — Executor (90–120 min)

Create `src/query/executor.py`. This is the most complex class — take it step by step.

```python
class Executor:
    def __init__(self, disk_manager, catalog):
        self.dm = disk_manager
        self.catalog = catalog

    def execute(self, stmt) -> list[tuple]:
        if isinstance(stmt, InsertStmt):
            return self._execute_insert(stmt)
        elif isinstance(stmt, SelectStmt):
            return self._execute_select(stmt)
        else:
            raise NotImplementedError(f"Unknown statement type: {type(stmt)}")

    def _execute_insert(self, stmt: InsertStmt) -> list[tuple]:
        table_info = self.catalog.get_table(stmt.table)
        schema = table_info['schema']
        ser = RecordSerializer(schema)
        row = tuple(stmt.values)

        record_bytes = ser.serialize(row)

        # Find a data page with free space
        page_id, page = self._find_writable_page(table_info)
        slot_id = page.add_record(record_bytes)
        self.dm.write_page(page_id, page.to_bytes())

        # Update the B-Tree index on the primary key
        pk_col = table_info['primary_key']
        pk_idx = [col for col, _ in schema].index(pk_col)
        pk_value = stmt.values[pk_idx]

        tree = BTree(self.dm, table_info['index_root_page'])
        tree.insert(pk_value, RecordPointer(page_id, slot_id))

        return []  # INSERT returns no rows

    def _execute_select(self, stmt: SelectStmt) -> list[tuple]:
        table_info = self.catalog.get_table(stmt.table)
        schema = table_info['schema']
        ser = RecordSerializer(schema)

        if stmt.where is not None:
            pk_col = table_info['primary_key']
            if stmt.where.column == pk_col:
                return self._index_scan(stmt, table_info, ser)

        return self._full_scan(stmt, table_info, ser)

    def _index_scan(self, stmt, table_info, ser) -> list[tuple]:
        tree = BTree(self.dm, table_info['index_root_page'])
        ptr = tree.search(stmt.where.value)
        if ptr is None:
            return []
        raw = self.dm.read_page(ptr.page_id)
        page = Page.from_bytes(ptr.page_id, raw)
        record_bytes = page.get_record(ptr.slot_id)
        return [ser.deserialize(record_bytes)]

    def _full_scan(self, stmt, table_info, ser) -> list[tuple]:
        results = []
        for page_id in table_info['data_pages']:
            raw = self.dm.read_page(page_id)
            page = Page.from_bytes(page_id, raw)
            for slot_id in range(page.num_slots):
                record = page.get_record(slot_id)
                if record is None:
                    continue  # deleted
                row = ser.deserialize(record)
                if self._matches_where(row, stmt.where, table_info['schema']):
                    results.append(row)
        return results

    def _matches_where(self, row, where, schema) -> bool:
        if where is None:
            return True
        col_names = [col for col, _ in schema]
        idx = col_names.index(where.column)
        return row[idx] == where.value
```

### Task 3.5 — Engine Facade and REPL (30 min)

Create `src/main.py`:

```python
from storage.disk_manager import DiskManager
from storage.catalog import Catalog
from query.lexer import Lexer
from query.parser import Parser
from query.executor import Executor

class Engine:
    def __init__(self, db_path: str):
        self.dm = DiskManager(db_path)
        self.catalog = Catalog(self.dm)
        self.executor = Executor(self.dm, self.catalog)

    def execute(self, sql: str) -> list[tuple]:
        tokens = Lexer(sql).tokenize()
        stmt = Parser(tokens).parse()
        return self.executor.execute(stmt)

    def close(self):
        self.dm.close()

if __name__ == '__main__':
    engine = Engine("data/database.db")
    print("db engine ready. type 'exit' to quit.")
    while True:
        try:
            sql = input("db> ").strip()
            if not sql:
                continue
            if sql.lower() in ('exit', 'quit'):
                break
            rows = engine.execute(sql)
            for row in rows:
                print(row)
        except Exception as e:
            print(f"Error: {e}")
    engine.close()
```

## 💡 Thinking Questions

1. What happens when you parse `SELECT * FROM users WHERE id = 'five'`? The WHERE value is a string but the column type is `int`. When should this error be caught — during parsing, during planning, or during execution?

2. Full table scans are O(n). Index scans are O(log n). When would a database *intentionally choose* a full scan over an index scan? (Hint: think about selectivity and the cost of random I/O.)

3. Your catalog stores table metadata in page 1 as JSON. What happens if two processes try to insert simultaneously and both try to update page 1? This is a preview of Phase 4.

4. `SELECT name FROM users WHERE id = 5` — the WHERE matches by index, giving you a record. But you only want `name`, not the full row. Does your current design return the full row or just the requested columns? What would need to change to project only the needed columns?

5. What is the difference between a query's *logical plan* (what to compute) and its *physical plan* (how to compute it)? In PostgreSQL, the planner produces a logical plan (e.g., "filter users by id") and the executor chooses the physical implementation (e.g., "use index scan with B-Tree on id"). Does your executor conflate these two concerns?

## ⚠️ Common Mistakes

**Mistake 1: Parser consuming tokens when it should peek**
Always use `_peek()` to check the next token before deciding what to parse, then `_consume()` to actually advance. Consuming too early means you skip tokens and the grammar breaks.

**Mistake 2: Catalog not persisted on every write**
If you `create_table()` but don't call `_save()`, the table exists in memory but not on disk. Next restart, it's gone. Call `_save()` at the end of every catalog-modifying method.

**Mistake 3: Executor assumes schema knows column positions**
Schema is `[("id", "int"), ("name", "str")]`. When you look up `WHERE name = 'alice'`, you need to find the index of `name` in the schema to compare the right field of a deserialized row. Hard-coding positions breaks immediately when column order changes.

**Mistake 4: Not handling the case where a data page fills up**
Your first insert works. Your 10,000th insert may fail because the first data page is full. `_find_writable_page` must allocate new pages. When it does, it must update `catalog.tables[name]['data_pages']` and call `_save()`.

**Mistake 5: Lexer not handling case-insensitive keywords**
`select` and `SELECT` are the same SQL keyword. Lowercase the identifier before looking it up in `KEYWORDS`. Identifiers (table/column names) should preserve case or be consistently lowercased — pick one and stick to it.

## 🔍 Debugging Guide

**Problem**: `SyntaxError: Expected FROM, got IDENT`  
**Cause**: Parser consumed the `FROM` keyword accidentally, or the token stream is out of order  
**Fix**: Print all tokens from the lexer before parsing. Check that `_consume(TokenType.FROM)` is not called twice.

**Problem**: `ValueError: Table users does not exist` after creating it  
**Cause**: Catalog `_save()` not called, or page 1 was not allocated before page 0's `num_pages` was set  
**Fix**: After `create_table`, close and reopen the engine. If the table is gone, your `_save()` is not being called.

**Problem**: `SELECT * FROM users` returns 0 rows after inserting  
**Cause**: `table_info['data_pages']` list is empty or contains wrong page IDs  
**Fix**: Print `catalog.tables` after every insert. Verify `data_pages` contains the page you wrote to.

**Problem**: Full scan returns duplicate or extra rows  
**Cause**: Iterating over more slots than actually exist, or not checking for tombstoned slots  
**Fix**: Iterate `for slot_id in range(page.num_slots)`, not a hardcoded range.
